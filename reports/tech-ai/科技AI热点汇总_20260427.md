## 📌 一、本日最重磅：微软-OpenAI 合作关系大重构 + AI 行业融资地震

### 1. 微软与 OpenAI 终结独家合作，开启"非排他"新时代 🔥🔥🔥
**[来源：Bloomberg / CNBC / Techmeme / HN 742🔥]**

微软和 OpenAI 于4月27日宣布重新修订合作协议，这是自2019年双方建立合作以来最大幅度的关系调整：

- **云排他终结**：OpenAI 现在可以在任何云平台（AWS、Google Cloud 等）上提供其全部产品，Azure 不再是唯一分发渠道，但微软仍为"首选云合作伙伴"（primary cloud partner），OpenAI 产品仍将优先在 Azure 上线
- **收入分成调整**：OpenAI 继续以20%的比例向微软支付收入分成，但设有总额上限，支付将于2030年终止；微软方面不再向 OpenAI 支付收入分成
- **IP 许可变为非排他**：微软对 OpenAI 模型和产品的 IP 许可延续至2032年，但从排他变为非排他
- **AGI 条款移除**：此前争议极大的"AGI 条款"（一旦 OpenAI 实现 AGI，微软可获得 IP 权利）被正式移除
- **背景**：微软自2019年起已向 OpenAI 投入超过$130亿，持有其营利性实体约27%的股份；OpenAI 此前已承诺在 Azure 上消费$2500亿

**💡 影响分析**：这笔交易为 OpenAI 打开了多云战略的大门。AWS CEO Andy Jassy 已宣布 OpenAI 模型将在 Bedrock 上线。对于推荐系统工程师来说，这意味着未来在不同云平台上部署 OpenAI 模型做 embedding/ranking 将更加灵活。

### 2. GitHub Copilot 转向按量计费，6月1日生效 🔥🔥🔥
**[来源：GitHub Blog / HN 538🔥 412评论]**

GitHub 宣布所有 Copilot 计划将于2026年6月1日从"Premium Request Units"（PRU）转为"GitHub AI Credits"按量计费：

| 计划 | 月费 | 含 AI Credits |
|------|------|--------------|
| Pro | $10/月 | $10 额度 |
| Pro+ | $39/月 | $39 额度 |
| Business | $19/用户/月 | $19 额度（6-8月促销期$30） |
| Enterprise | $39/用户/月 | $39 额度（6-8月促销期$70） |

- **代码补全和 Next Edit 建议仍然免费**，不消耗 credits
- 用量按 token 消耗计算（输入、输出、缓存 token），费率跟随各模型 API 定价
- 新增多级预算控制（企业、成本中心、用户级别）
- 5月初上线预览账单体验，让用户提前了解预估成本
- **年度订阅用户**在当前计划到期前保持原有 PRU 计费不变

> **HN 社区情绪**：讨论激烈（412条评论），主要担忧是"按量计费"可能导致大型 agentic 编码会话成本飙升，社区对 GitHub 此举褒贬不一。

### 3. Mercor 遭遇 4TB 语音数据泄露，4万名 AI 承包商受影响 🔥🔥🔥
**[来源：Oravys / HN 434🔥 160评论]**

AI 众包招聘平台 Mercor（估值$100亿，服务 OpenAI、Google DeepMind、Meta 等）遭遇严重数据泄露：

- **泄露规模**：4TB 语音样本 + 政府签发的身份证件
- **受影响人数**：约40,000名 AI 承包商
- **攻击者**：Lapsus$ 勒索组织，数据已被发布到暗网泄露站点
- **攻击向量**：通过 LiteLLM 代理供应链攻击（poisoned proxy）入侵
- **风险**：泄露的语音样本平均2-5分钟/人，远超高质量语音克隆所需的15秒阈值；可用于绕过多因素身份验证、银行电话验证、deepfake 视频通话
- **后果**：Meta 已暂停与 Mercor 的合作，多起诉讼正在准备中

**💡 安全警示**：FBI 报告显示2026年仅冒充语音电话诈骗就造成$23亿损失。Pindrop 数据显示2025年保险呼叫中心合成语音攻击增长475%。这是 AI 时代生物特征数据泄露的标志性事件。

Microsoft remains OpenAI's primary cloud partner `[primary cloud partner: 首选云合作伙伴——注意"primary"不等于"exclusive"（排他性），这正是本次变更的关键]`, with OpenAI products shipping first on Azure `[shipping first on: 优先在……上发布——"ship"在科技语境中是"发布/上线"的意思]` unless Microsoft is unable to support necessary capabilities `[necessary capabilities: 必要功能——委婉表达"除非微软跟不上技术需求"]`. OpenAI can now serve all of its products across any cloud provider `[serve products across any cloud provider: 在任何云平台上提供其产品——"serve"此处是"提供服务"的意思，而非"服务于"]`.

Microsoft's license `[license: 许可/授权——法律用语，指使用知识产权的正式权利]` to OpenAI IP for models and products extends through 2032 and is now non-exclusive `[non-exclusive: 非排他性的——这是本次变更最重要的法律术语变化，意味着其他公司也可获得类似许可]`.

Microsoft will no longer pay revenue share `[revenue share: 收入分成——"share"此处是名词，表示分成比例]` to OpenAI. OpenAI continues to pay revenue share to Microsoft through 2030, at the same percentage, with a total cap `[cap: 上限/封顶——商业合同中极常见，指设定最高限额]`, independent of OpenAI's technology progress `[independent of: 不受……影响/与……无关——暗指取消了之前与AGI实现挂钩的条款]`.

Microsoft continues as a major shareholder `[shareholder: 股东——"major shareholder"即大股东]` in OpenAI's growth. We continue our work together on datacenter capacity `[datacenter capacity: 数据中心容量/算力——capacity在此指计算和存储能力]`, next-generation silicon collaboration `[silicon collaboration: 芯片合作——"silicon"代指芯片/半导体]`, and AI cybersecurity `[cybersecurity: 网络安全——AI安全是当前热门交叉领域]` applications.

| 短语 | 含义 | 用法 |

| grounded in | 以……为基础 | 用于正式声明，表达核心价值观或原则 |
| meet enterprises where they are | 适配客户现有环境 | 商业语境中表示去找客户而非让客户来找你 |

| independent of | 不受……影响 | 法律/商业文本中常见的条件限定表达 |

### 技术写作模式
1. **公告体（Announcement Style）**：微软博文使用大量被动语态和名词化表达（"announcing an amended agreement"而非"we changed our deal"），这是企业公告的标准写法——正式、客观、降低情感色彩。
2. **对比递进**：CNBC 报道巧妙使用"Things have changed since..."开篇，通过时间对比（Oct 2025 → Apr 2026）构建叙事张力，这是商业新闻的常见结构。
3. **数字锚定**：文中多次用精确数字（$13 billion, 27%, 20%, 2030, 2032）锚定读者注意力——学习英文商业写作时注意数字的战略性使用。

### 语法亮点
1. **独立主格结构**："with OpenAI products shipping first on Azure" —— "with + 名词 + 现在分词"构成独立主格，做伴随状语，表示附加信息。这种结构在正式写作中非常常见。
2. **非限制性关系从句**："which is 20%" —— "which"引导的非限制性从句补充说明前面的"same percentage"，用逗号隔开，提供补充信息而非限定。
3. **条件让步**："unless Microsoft is unable to support necessary capabilities" —— "unless"引导条件从句，表达例外情况。商业合同和公告中大量使用这种"规则+例外"结构。
