## 📌 一、本日最重磅：Axios 供应链攻击 & Claude Code 源码泄露——安全风暴席卷开发者社区

### 1. Axios npm 包遭供应链攻击，植入跨平台远程木马 🔥🔥🔥
**[来源：HN 1777🔥 / Techmeme / Snyk / The Hacker News]**

流行 HTTP 客户端 Axios（npm 周下载量超 1 亿次）遭遇严重供应链攻击：

- **受影响版本**：`axios@1.14.1` 和 `axios@0.30.4`，于 3 月 31 日 00:21–01:00 UTC 之间发布
- **攻击手法**：攻击者劫持了 Axios 首席维护者 `@jasonsaayman` 的 npm 账号，向 `package.json` 注入恶意依赖 `plain-crypto-js@4.2.1`
- **恶意负载**：通过 `postinstall` 钩子执行双重混淆（Reversed Base64 + XOR 加密）的 dropper，自动检测操作系统并下载第二阶段 RAT（远程访问木马）
  - **macOS**：伪装为 Apple 守护进程（`/Library/Caches/com.apple.act.mond`），每 60 秒回连 C2 服务器，可注入二进制文件、运行 shell 命令
  - **Windows**：伪装为 Windows Terminal（`%PROGRAMDATA%\wt.exe`），绕过执行策略运行 PowerShell RAT
  - **Linux**：通过 `nohup python3` 以孤儿后台进程形式运行
- **归因**：Google 将此次攻击归咎于朝鲜威胁组织 **UNC1069**
- **影响范围**：恶意版本在线约 3 小时后被移除，但在此期间所有运行 `npm install` 的 CI/CD 流水线和开发机器均有感染风险
- **补救措施**：立即审计 lockfile、轮换密钥凭证、隔离受感染系统

> **⚠️ 对推荐/广告系统工程师的提醒**：如果你的模型服务或特征工程管线使用了 Node.js 微服务（如 BFF 层），务必立即检查 Axios 版本。

### 2. Anthropic Claude Code 完整源码泄露——50 万行代码公开 🔥🔥🔥
**[来源：HN 754🔥 / Techmeme / Fortune / aiHola]**

Anthropic 因 npm 打包错误，意外将 Claude Code 完整源代码随 `cli.js.map` 文件发布：

- **泄露规模**：约 **1,900 个 TypeScript 文件、512,000 行代码**
- **原因**：Bun 打包器默认生成源映射（source map），Anthropic 既未配置打包器跳过生成，也未将 `*.map` 添加到 `.npmignore`
- **这是第二次**：2025 年 2 月 Claude Code 发布当天就发生过相同泄露，此后发布了 363 个版本仍未修复
- **泄露的隐藏功能**（44 个功能标志）：
  - **KAIROS**：持久助手模式（"Always-On Claude"），跨会话维护上下文，包含夜间"做梦"信息整合流程
  - **ULTRAPLAN**：将复杂规划任务卸载到运行 Opus 4.6 的远程云容器，最长 30 分钟思考时间
  - **BUDDY**：电子宠物系统，18 个物种、稀有度等级、五大属性（调试、耐心、混乱、智慧、吐槽）
  - **卧底模式**（Undercover Mode）：专门设计用于从 git 提交中清理内部信息——讽刺的是，源代码本身被泄露了
- **新模型线索**：代码引用了 **Mythos**（内部代号 Capybara），定位为比 Opus 更强大的新层级
- **安全影响**：权限模型、认证流程、工具审批逻辑全部公开，竞争对手和安全研究者正在大量研究

## 精读原文（含生词/短语/语法标注）

On March 31, security researcher Chaofan Shou posted six words on X that Anthropic would probably pay good money to `unsee` [unsee: 无法"取消看到"——这里是创造性用词，表示 Anthropic 宁愿这些话从未出现]: "Claude code source code has been leaked via a map file in their npm registry." Within hours the post had over 4.5 million views. Multiple GitHub mirrors appeared with the full, `neatly extracted` [neatly extracted: 整洁地提取出来的——neat 指干净利落] codebase. And Anthropic, the AI company that built its brand on being `the careful one` [the careful one: 那个小心谨慎的——用定冠词+形容词做名词化表达，指代"以谨慎著称的公司"], was busy `unpublishing` [unpublish: 取消发布——npm 术语，指撤回已发布的包版本] a package version for the second time in thirteen months.

The mechanism was simple enough to be `insulting` [insulting: 侮辱性的——这里形容漏洞太低级让人觉得被侮辱]. Version 2.1.88 of the npm package shipped with a 59.8-megabyte `cli.js.map` file containing the full, `unobfuscated` [unobfuscated: 未混淆的——obfuscate 意为混淆、使难以理解，un- 前缀取反] TypeScript source for Claude Code. The source map's internal references pointed to an R2 cloud `bucket` [bucket: 桶/存储桶——云存储术语，指对象存储的容器] on Anthropic's own infrastructure, where the entire `src/` directory sat as a downloadable zip. Anyone who ran `npm install` could find it in their `node_modules` folder.

**It has happened before**

On February 24, 2025—the day Claude Code launched—developer Dave Shoemaker opened the bundled `cli.mjs` in Sublime Text and found an 18-million-character inline source map `encoded as base64` [encoded as base64: 以 base64 编码——常见的二进制转文本编码方式]. Anthropic quietly pulled it, `purged` [purged: 清除——比 delete 更彻底，有"净化"的含义] it from npm caches, and moved on. Shoemaker only recovered the original because Sublime Text's undo history still held it.

Between that incident and this one, Anthropic shipped 363 versions of the package. At some point the source map `crept back in` [crept back in: 悄悄溜了回来——creep 的过去式，形象地描述问题悄然复现]. Not inline this time, but as a separate `.map` file sitting right next to `cli.js`. Nobody at Anthropic caught it.

Claude Code uses `the Bun bundler` [Bun bundler: Bun 打包工具——Bun 是新一代 JavaScript 运行时], which generates source maps by default. Excluding them requires either configuring the bundler to skip generation or adding `*.map` to an `.npmignore` file. Neither happened. And here's the part that's `genuinely hard to process` [genuinely hard to process: 真的很难理解/接受——比 hard to believe 更有力度]: the leaked codebase contains a subsystem called "undercover mode" specifically designed to `scrub` [scrub: 擦洗/清除——安全领域术语，指从数据中移除敏感信息] internal Anthropic information from git commits to prevent `leakage` [leakage: 泄露——名词形式] into public repositories. They built a system to prevent leaks. Then they shipped the source through a `.map` file.

**What's actually inside**

The extracted codebase spans roughly 1,900 TypeScript files and 512,000 lines. Claude Code runs on Bun (not Node), uses React and Ink for terminal rendering, and `architecturally speaking` [architecturally speaking: 从架构上讲——常见的科技写作过渡短语], it's considerably more `sophisticated` [sophisticated: 精密复杂的——不仅指"高级"，更强调设计的精巧程度] than `what you'd expect from a polished CLI` [what you'd expect from a polished CLI: 你对一个精致 CLI 的期望——这里暗示实际复杂度远超表面]. The main entry point alone is 785KB.

The tool system contains roughly 40 `discrete` [discrete: 离散的/独立的——不要和 discreet（谨慎的）混淆] tools across 29,000 lines handling everything from file reads to bash execution to `sub-agent spawning` [sub-agent spawning: 子代理生成——spawn 在计算机领域指创建/启动新进程]. Every tool is `permission-gated` [permission-gated: 受权限门控的——gate 作动词表示"设置门槛/关卡"]. The query engine has 46,000 lines managing all LLM API calls, streaming, caching, and `orchestration` [orchestration: 编排——分布式系统术语，指协调多个组件协同工作]. It's the largest module in the codebase.

But the unreleased features are what `lit up` [lit up: 点燃了——这里指引爆了讨论热度] Hacker News and Reddit. The code contains 44 `feature flags` [feature flags: 功能标志——软件工程术语，用编译时或运行时开关控制功能是否启用] covering capabilities that are fully built but set to "false" `via compile-time toggles` [via compile-time toggles: 通过编译时开关——toggle 指切换开关] in external builds.

**KAIROS, BUDDY, and the stuff Anthropic wasn't ready to talk about**

**KAIROS** is a `persistent` [persistent: 持久的——指跨会话维持状态，不会随终端关闭而消失] assistant mode described in the code as "Always-On Claude." It maintains context across sessions, stores memory logs in a private directory, and includes a `nightly "dreaming" process` [nightly "dreaming" process: 夜间"做梦"过程——隐喻性命名，指后台信息整合] that `consolidates` [consolidates: 整合——常见于数据处理和机器学习领域] information while the user sleeps. The code handles midnight `boundary conditions` [boundary conditions: 边界条件——工程术语，指特殊或极端情况的处理] so the dreaming cycle doesn't interrupt. It has access to tools that regular Claude Code doesn't and uses a special "briefing" output mode designed for a persistent assistant that `shouldn't flood your terminal` [shouldn't flood your terminal: 不应淹没你的终端——flood 作动词表示大量涌入].

**ULTRAPLAN** `offloads` [offloads: 卸载/转移——将计算任务从本地转到远程] complex planning tasks to a remote cloud container running Opus 4.6, gives it up to 30 minutes of thinking time, and lets users approve the result from a browser. There's a `sentinel value` [sentinel value: 哨兵值——编程术语，用于标记特殊状态的特定值] called `__ULTRAPLAN_TELEPORT_LOCAL__` that `teleports` [teleports: 传送——借用科幻概念描述远程结果传回本地的过程] results back to the local terminal. I don't know what to make of the name.

Then there's **BUDDY**. `A Tamagotchi-style` [Tamagotchi-style: 拓麻歌子/电子宠物风格——Tamagotchi 是 90 年代日本经典电子宠物] AI pet that sits in a speech bubble next to your prompt. Eighteen species (duck, dragon, axolotl, capybara, mushroom, ghost), `rarity tiers` [rarity tiers: 稀有度等级——游戏术语] from common to 1% legendary, cosmetics including hats and `shiny variants` [shiny variants: 闪光变体——源自宝可梦的"色违/闪光"概念], five stats: DEBUGGING, PATIENCE, CHAOS, WISDOM, SNARK. Claude generates the name and personality on first `hatch` [hatch: 孵化——这里指电子宠物首次生成]. According to the leaked code, the plan was a `teaser rollout` [teaser rollout: 预告式推出——teaser 指吊胃口的预告] April 1–7, May launch, `starting with Anthropic employees` [starting with Anthropic employees: 从 Anthropic 员工开始——内部先行测试的策略].

Someone at Anthropic was having fun.

**The pattern problem**

This is Anthropic's second leak in five days. On March 26 *Fortune* reported that a CMS misconfiguration exposed nearly 3,000 unpublished assets, including draft blog posts describing an unreleased model called Claude Mythos (internal product name: Capybara, which also explains the pet species). That leak revealed details of a model that Anthropic itself warned carries "`unprecedented cybersecurity risk`" [unprecedented cybersecurity risk: 前所未有的网络安全风险——unprecedented 指历史上从未有过的]. The company blamed "human error in a CMS configuration."

Five days later: human error in an npm configuration.

Anthropic's `spokesperson` [spokesperson: 发言人] statement after the CMS incident carefully noted that AI tools were `not at fault` [not at fault: 没有过错——at fault 是固定搭配，表示负有责任]. The exposed material was "early drafts" and did not involve "core infrastructure, AI systems, customer data, or security architecture." The npm leak is harder to `characterize` [characterize: 定性/描述——法律和公关语境中常用，比 describe 更正式] that way. This is actual source code for a production tool that has file-system access, terminal access, and codebase access on developer machines. The permission model, auth flows, and tool approval logic are now public. As one Hacker News commenter put it, "A company you trust with file system access failed to properly secure its own software."

Others are less concerned. Source maps were technically extractable since launch, and the argument goes that `client-side CLI code` [client-side CLI code: 客户端 CLI 代码——强调这是运行在用户机器上的代码，不是服务端] is different from model weights or training data. `It's the slot machine, not the house edge` [It's the slot machine, not the house edge: 这是老虎机，不是庄家优势——赌场比喻，slot machine（老虎机）是前端展示层，house edge（庄家优势/赌场利润率）才是核心秘密]. Fair point. But the slot machine's permission system is the barrier between Claude and your file system.

**What happens next**

Anthropic has pulled the affected package version and pushed an update without source maps. But the code has been mirrored across multiple GitHub repos, forked hundreds of times, and is being actively studied. Whether Anthropic will pursue `takedown` [takedown: 删除/下架——法律术语，常见于 DMCA takedown（版权删除通知）] notices is unclear. The architecture patterns, tool system, `multi-agent orchestration` [multi-agent orchestration: 多代理编排——AI 系统架构术语]—all of it is now part of the public conversation about how to build agentic AI applications.

The timing `compounds` [compounds: 加剧——compound 作动词表示使问题叠加/恶化] the embarrassment. Anthropic is `gearing up for` [gearing up for: 正在为……做准备——gear up 指准备就绪] an IPO. Its competitor OpenAI just finished `pre-training` [pre-training: 预训练——LLM 训练的第一阶段] its own model (codenamed Spud). And Anthropic's brand—the thing that `sets it apart` [sets it apart: 使其与众不同——set apart 是固定搭配] in the AI race—is that it thinks more deeply about safety and risk than anyone else. Two configuration mistakes in five days don't destroy that argument, but they do make the next safety briefing a little awkward.

One reply on Shou's original post summed it up: "Nothing says `'agentic future'` [agentic future: 智能体的未来——agentic 是 AI 领域新造的形容词，表示具有自主代理能力的] quite like accidentally publishing your source."

### 技术写作模式
1. **数字具象化**："512,000 lines of code"、"59.8-megabyte"、"363 versions"——好的科技写作用精确数字建立可信度，避免模糊的"大量""很多"
2. **对比讽刺法**："They built a system to prevent leaks. Then they shipped the source through a .map file."——通过并列两个矛盾事实制造讽刺效果，是科技评论的常用手法
3. **赌场比喻延展**："It's the slot machine, not the house edge. Fair point. But the slot machine's permission system is the barrier between Claude and your file system."——先引用对方论点，承认合理性（Fair point），再用同一比喻反驳，论证有力且优雅
4. **人物引语收尾**：文章以一条 X 平台回复结尾——让读者的声音成为结语，增强共鸣感

### 语法亮点
1. **独立主格结构**："And Anthropic, the AI company that built its brand on being the careful one, was busy unpublishing a package version..."——用同位语从句（the AI company that...）插入背景信息，主句被优雅地推后
2. **关系代词省略**："the thing that sets it apart" 中 that 引导定语从句修饰 thing；"the argument goes that..."中 the argument goes 作为插入语增加权威性
3. **形容词名词化**："the careful one"——用 the + 形容词 + one 指代某个实体，比直接用名字更有修辞效果
4. **虚拟/反事实**："would probably pay good money to unsee"——would + 动词表示假设情景，这里暗示 Anthropic 如果能回到过去一定会阻止这件事
