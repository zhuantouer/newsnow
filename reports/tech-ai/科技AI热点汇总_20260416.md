## 📌 一、本日最重磅：Anthropic Opus 4.7 发布 + OpenAI Codex 全面进化 + 阿里 Qwen3.6 开源

### 1. Anthropic 正式发布 Claude Opus 4.7——"Mythos 民用版"来了 🔥🔥🔥
**[来源：Anthropic 官方 / Techmeme / HN（1455🔥）/ CNBC / VentureBeat / Gizmodo]**

Anthropic 发布最新旗舰模型 Claude Opus 4.7，称其在高级软件工程方面相比 Opus 4.6 有"显著改进"，但坦承"不如 Mythos Preview 那样全面"。

**核心升级：**
- **SWE-Bench Verified 87.6%**（+7%），SWE-Bench Pro 64.3%（+11%），TerminalBench 2.0 达 69.4%（+4%）
- **CursorBench 从 58% 跳至 70%**，Rakuten-SWE-Bench 解题量为 Opus 4.6 的 3 倍
- **XBOW 视觉准确率 98.5%**（vs Opus 4.6 的 54.5%），支持 3.3 倍分辨率图像输入（最高 2576px 长边）
- 新增 **xhigh 推理级别**（介于 high 和 max 之间），Claude Code 默认提升至 xhigh
- 新分词器（tokenizer），同一输入可能映射为 1.0-1.35× tokens
- **定价不变**：$5/$25 每百万 token（输入/输出）
- 已上线 API、Amazon Bedrock、Vertex AI、Microsoft Foundry

**Claude Code 同步更新：**
- 新增 `/ultrareview` 命令——专门的代码审查会话，Pro/Max 用户可免费试用 3 次
- Max 用户解锁 **Auto Mode**——Claude 自主决策运行，减少中断
- 可重复例程（Repeatable Routines）自动化更新

**争议焦点：**
- Ethan Mollick（@emollick）批评其自适应思维机制——非代码/数学任务常被判定"低难度"，结果质量反降
- Gizmodo 标题直接嘲讽："Opus 4.7 的发布只是为了提醒大家 Mythos 有多强"
- @hosseeb："我们正式进入 AI 的不民主时代——你不够重要，无法获得最强智能"
- **网络安全能力被刻意削弱**——Anthropic 承认在训练中"实验性降低了这些能力"，Mythos 仍为限量发行

**zishuai 视角 🎯**：Opus 4.7 的 Rakuten-SWE-Bench 3x 提升和 XBOW 视觉 98.5% 表明多模态 Agent 能力已达工业可用水平。xhigh effort level 的引入对需要精确推理的广告推荐场景（如复杂 CTR 模型调参）可能有实际价值。

## 原文精读（含标注）

### INTRODUCING CLAUDE OPUS 4.7

Our latest model, Claude Opus 4.7, is now **generally available** `[generally available (GA): 正式发布/全面可用——软件行业标准术语，区别于 preview/beta]`.

Opus 4.7 is a **notable improvement** `[notable: 值得注意的、显著的——比 significant 更克制的表达]` on Opus 4.6 in advanced software engineering, with particular gains on the most difficult tasks. Users report being able to **hand off** `[hand off: 交接、移交——常用于工作委派场景，"hand off work to sb"]` their hardest coding work—the kind that previously needed **close supervision** `[close supervision: 密切监督——close 在此意为"紧密的"而非"关闭"]`—to Opus 4.7 with confidence. Opus 4.7 handles complex, **long-running tasks** `[long-running tasks: 长时间运行的任务——Agent 领域高频术语]` with **rigor** `[rigor: 严谨性、缜密——学术和工程语境中的高频词]` and consistency, pays precise attention to instructions, and **devises** `[devise: 设计、想出（方法/计划）——比 think up 更正式]` ways to **verify its own outputs** `[verify its own outputs: 验证自身输出——self-verification 是当前 Agent 研究热点]` before reporting back.

The model also has **substantially** `[substantially: 大幅地、本质上——比 significantly 更强调程度]` better vision: it can see images in greater resolution. It's more **tasteful** `[tasteful: 有品味的、高雅的——用于形容 AI 的审美能力是新用法]` and creative when completing professional tasks, producing higher-quality interfaces, slides, and docs. And—although it is **less broadly capable** `[less broadly capable: 综合能力不如——委婉表达"不如 Mythos 强"]` than our most powerful model, Claude Mythos Preview—it shows better results than Opus 4.6 across a range of **benchmarks** `[benchmark: 基准测试——AI 领域核心术语]`.

Last week we announced **Project Glasswing** `[Project Glasswing: Anthropic 的网络安全 AI 项目代号]`, highlighting the risks—and benefits—of AI models for cybersecurity. We stated that we would keep Claude Mythos Preview's release limited and test new **cyber safeguards** `[cyber safeguards: 网络安全防护措施]` on less capable models first. Opus 4.7 is the first such model: its cyber capabilities are not as advanced as those of Mythos Preview (indeed, during its training we **experimented with efforts to differentially reduce these capabilities** `[differentially reduce: 差异化地削减——关键表述：有选择性地降低特定能力而非全面降低]`). We are releasing Opus 4.7 with safeguards that automatically detect and block requests that indicate **prohibited or high-risk cybersecurity uses** `[prohibited or high-risk uses: 被禁止的或高风险用途——合规用语]`. What we learn from the **real-world deployment** `[real-world deployment: 真实世界部署——区别于实验室测试]` of these safeguards will help us work towards our **eventual goal** `[eventual goal: 最终目标——eventual 强调"终将到来的"]` of a **broad release** `[broad release: 广泛发布——与 limited release 对应]` of Mythos-class models.

### MIGRATING FROM OPUS 4.6 TO OPUS 4.7

Opus 4.7 is a **direct upgrade** `[direct upgrade: 直接升级——可无缝替换]` to Opus 4.6, but two changes are worth **planning for** `[worth planning for: 值得提前规划——比 worth noting 更强调需要行动]` because they affect token usage. First, Opus 4.7 uses an **updated tokenizer** `[tokenizer: 分词器——将文本切分为 token 的组件]` that improves how the model processes text. The tradeoff is that the same input can map to more tokens—roughly **1.0–1.35×** depending on the content type. Second, Opus 4.7 **thinks more at higher effort levels** `[thinks more: 思考更多——extended thinking 的通俗表达]`, particularly on **later turns in agentic settings** `[later turns in agentic settings: Agent 场景中的后续轮次——Agent 多步推理]`. This improves its reliability on hard problems, but it does mean it produces more output tokens.

Users can control token usage in various ways: by using the **effort parameter** `[effort parameter: 推理努力参数]`, adjusting their task budgets, or **prompting the model to be more concise** `[prompting the model to be more concise: 通过提示词让模型更简洁——prompt engineering 实用技巧]`.

### 技术写作模式
1. **"委婉承认不足" 模式**：Anthropic 用 "less broadly capable than Mythos" 而非 "weaker than Mythos"——技术博客中承认产品局限性的标准话术，保持专业感同时管理预期。
2. **"客户证言矩阵" 模式**：按不同使用场景（编码/金融/法律/安全）排列客户评价，每条都精确到数字（+13%、3x、98.5%），构建全方位的社会证明。
3. **"权衡透明度" 模式**：明确告知升级代价（tokenizer 变化导致 1.0-1.35× token 消耗），而非隐藏——这在 AI 公司的产品文档中越来越常见，建立信任。
4. **"动作动词密集" 模式**：devises、garners、lifts、resists、unlocks——避免被动语态，让技术描述更有力量感。

### 语法亮点
1. **破折号插入语**："the kind that previously needed close supervision—to Opus 4.7 with confidence"——用双破折号插入补充说明，比括号更流畅，在技术博客中极为常见。
2. **让步状语从句前置**："And—although it is less broadly capable than Mythos Preview—it shows better results"——although 从句被破折号包裹并前置，兼具让步和强调效果。
3. **动名词短语做主语**："What we learn from the real-world deployment... will help us work towards..."——what 引导的名词性从句做主语，常用于论述因果关系。
4. **分号连接并列句**："In others, Opus 4.7 is modestly weaker; on some measures, it is an improvement"——分号用于连接对比关系的两个独立句，比 but 更正式。
